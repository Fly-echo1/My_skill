# 嵌入式开发模式 Embedded Dev

## 概述

这是一个**嵌入式开发专用初始化流程**。调用后自动完成：
1. 加载 Superpowers 核心技能（brainstorming、writing-plans、verification-before-completion 等）
2. 连接 WeChat 推送通道，每个里程碑自动通知到手机
3. 引导你完成 需求澄清 → 制定计划 → 子Agent驱动开发 的完整流程
4. 全程实时汇报进度（聊天窗口 + 微信双通道）

## 初始化流程

当你输入「开始嵌入式开发」或调用本 skill 时，执行以下步骤：

### 步骤 1：连接微信通知通道

主动询问用户：**「开发过程需要通过微信实时推送进度到手机吗？」**

- 如果用户同意：
  1. 执行 `mcp__claw__config` action=`add_channel` type=`wechat`
  2. 获取到微信二维码图片后，展示给用户扫码
  3. 等待用户完成扫码绑定
  4. 绑定成功后提示：✅ 微信通道已连接，后续里程碑将自动推送到手机
- 如果用户拒绝或已有微信通道 → 跳过此步，聊天窗口汇报即可

### 步骤 2：确认项目上下文

主动询问用户：
- 项目/任务描述（做什么？）
- 目标 MCU/平台（STM32？ESP32？Linux？）
- 开发语言（C？C++？Rust？）
- 约束条件（功耗？实时性？内存限制？）

### 步骤 3：启动 brainstorming

调用 Superpowers 的 brainstorming 流程，产出需求摘要，让用户确认。

### 步骤 4：创建 PLAN.md

根据确认的需求，调用 writing-plans 产出分步实施计划。

### 步骤 5：启动 subagent-driven-development

按计划执行，**全程遵守以下实时汇报规则**。

## 实时汇报规则（关键）

执行过程中**每发生一个里程碑事件**都要同步汇报：

### 聊天窗口汇报格式（必须）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 [任务 X/Y] 任务名称
   ├─ ▶ 派发实施子 Agent...（模型名）
   ├─ 💬 [子Agent提问] 具体问题
   │   └─ ⏳ 等待你的回复...
   ├─ ✅ 实施完成
   ├─ ▶ 派发规范审查子 Agent...
   ├─ ✅ 规范审查通过
   ├─ ⚠ [或] ❌ 规范审查发现问题：...
   ├─ ▶ 派发质量审查子 Agent...
   ├─ ✅ 质量审查通过
   └─ 🔄 进入下一个任务...
━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 微信推送规则（必须）

使用 `mcp__claw__notify` 推送以下场景（不刷屏）：

| 触发事件 | 推送内容 | 示例 |
|---|---|---|
| 子Agent提问 | 💬 问题 + 任务名 | `💬 [UART驱动] 波特率可配置还是固定？` |
| 规范审查不通过 | ❌ 不通过原因 | `❌ [UART驱动] 规范审查：缺少错误处理` |
| 质量审查不通过 | ⚠ 问题清单 | `⚠ [环形缓冲区] 质量审查：魔法数字` |
| 任务完成 | ✅ 任务完成 | `✅ [UART驱动] 任务完成，进入下一项` |
| 全部完成 | 🎉 全部完成 | `🎉 全部 X 个任务完成！` |

不要推送：常规进度（开始实施、审查通过等），这些在聊天窗口看即可。

## 使用的工具

- `mcp__claw__notify` — 微信推送
- `Agent` — 派发子 Agent（带 `run_in_background` 以异步跟踪）
- `Skill` — 调用 installed Superpowers skills

## 子 Agent 执行说明

- 每个子 Agent 通过 `Agent` 工具派发
- 派发时在 prompt 中要求子 Agent 在关键节点主动报告状态
- 使用 `run_in_background` + 监听通知来异步跟踪进度
- 收到子 Agent 完成通知后立即执行 review 并汇报

## 注意事项

- 子 Agent **不能并行派发实施任务**（写冲突），串行执行
- 审查子 Agent 可以与实施串行（一个完成了再下一个）
- 如果子 Agent 提问，必须等用户回复后再继续
- 所有代码改动前先确认不在 main/master 分支上
